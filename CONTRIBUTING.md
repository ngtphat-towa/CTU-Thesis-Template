# Contributing to CTU Thesis Template

Thank you for your interest in contributing to the CTU Thesis Template! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs
If you find a bug, please create a new issue using the [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md).
- **Check for existing issues** to avoid duplicates.
- **Describe the bug** clearly, including steps to reproduce.
- **Include screenshots** or error logs if applicable.

### Requesting Features
If you have an idea for a new feature, please use the [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md).
- Explain **why** the feature is needed.
- Describe **how** it should work.

### Submitting Changes (Pull Requests)
1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  **Create a new branch** for your feature or fix (`git checkout -b feature/my-new-feature`).
4.  **Make your changes**. Ensure you follow the project's coding style (Typst).
5.  **Test your changes**. Generate a PDF to verify everything looks correct.
6.  **Commit** your changes with clear messages.
7.  **Push** to your fork (`git push origin feature/my-new-feature`).
8.  **Open a Pull Request** to the `main` branch of the original repository.

## Style Guide
- Use **Typst** for all template logic and styling.
- Keep the `info.typ` structure clean and documented.
- Comments are encouraged for complex logic.
- Follow the official [CTU Thesis Guidelines](docs/CTU_THESIS_GUIDELINES.md).

## Questions?
If you have any questions, feel free to open a [Discussion](https://github.com/ngtphat-towa/CTU-Thesis-Template/discussions) or an Issue.
